### 1. [Wispr Flow — Dictation That Works Everywhere](https://wisprflow.ai/) (PH ▲527, 当日榜首)
- **是什么**: 系统级语音听写, 跨所有 Mac/PC 应用, 比打字快 4x。不是 voice assistant, 是 "stop typing, start speaking" 的纯输入层。
- **差异化**: vs Whisper / Otter / 内建 dictation — 强调 **everywhere**(任何应用任何输入框)+ **零延迟感**。打字习惯被"语音 + 自动整理"替代是产品级押注。
- **技术/打法**: 推断: 端侧 STT + 后处理(去口癖、加标点、改语序), 跨应用通过系统输入法/IME 钩子。变现走个人订阅 + 企业。
- **PM takeaway**: 如果做 multi-agent / 群体智能(像 [Project-A]/[Project-B]), 用户给 agent 喂 prompt 的方式 12 个月内会从打字切到说话, **prompt UX 该重新设计**(短句、口语化容错、打断重说)。
- **🎯 今天具体动作**: 装一下 Wispr Flow 试用 30 分钟, 体验 voice→prompt 的延迟与改写质量, 决定 [Project-B] 输入层是否提前预留 voice channel。
- **延伸**: 同日 PH 第 19 [Realtime TTS-2 by Inworld](https://inworld.ai/) (▲5) 在做生成侧, 一收一发刚好闭环。

### 3. [Chrome 静默安装 4GB Gemini Nano](https://www.thatprivacyguy.com/blog/chrome-silent-nano-install/) (HN ▲1464, 当日最高热)
- **是什么**: Chrome 自 2026-04-24 起在用户毫无知情下下载 4GB Gemini Nano 到本地, 删了会重装。
- **差异化**: 不是新模型, 是新的**分发姿态** — 浏览器变成 vendor 的 deployment surface, 绕过用户许可装客户端 AI。
- **技术/打法**: built-in AI 落地。模型常驻, Chrome 子系统可零延迟调用, 免 server roundtrip。
- **PM takeaway**: 浏览器/OS 层正在变成 AI 分发主战场。**对 multi-agent 产品**: 浏览器内置 LLM 后, agent 该跑在云端还是劫持浏览器内置模型? 后者**延迟、成本、隐私**全胜, 但要赌 Chrome API 开放。这是 12 个月内必须押的方向。
- **🎯 今天具体动作**: 查 Chrome built-in AI API 现状 (`chrome://flags` + `window.ai` API 文档), 估一下 [Project-B] 移植到浏览器内置模型的路径长度 — 写 1 页 spike doc 做 3 个月内 vs 12 个月内的决策时点。
- **延伸**: 同日 HN 第 8 [Gemma 4 multi-token prediction](https://blog.google/innovation-and-ai/technology/developers-tools/multi-token-prediction-gemma-4/) (583 pts) 是 Google 的另一手 — 推理加速。配合看 = Google 在端侧 AI 同时打速度和分发两条线。

### 4. [Cloudflare × Stripe Projects: Agent 自建账户、买域名、订阅](https://blog.cloudflare.com/agents-stripe-projects/) (HN ▲348)
- **是什么**: agent 用 Stripe Projects 做身份/支付编排, OAuth+OIDC+payment token, 自动开 Cloudflare 账户、注册域名、买 API token、部署应用。默认 $100/月限额。
- **差异化**: 第一次有大厂把"agent 当采购方"做成产品级协议, 不是 demo。三段式: service discovery → identity attestation → payment tokens。
- **技术/打法**: Stripe 抢"agent 钱包+身份"位置, Cloudflare 抢"agent 部署目的地"位置。共建协议而非各自 SDK。
- **PM takeaway**: 你的 SaaS/API/agent 平台, 有没有"agent purchase flow"? **如果 [Project-B]/[Project-A] 未来要被外部 agent host 调用**: signup、auth、billing、discovery 四个面要为 agent 设计第二套入口。
- **🎯 今天具体动作**: 列一份 [Project-B] 现有 4 个用户入口 (signup/auth/billing/discovery) 表格, 对照 Cloudflare-Stripe 协议三段式, 标出"已 agent-ready / 需改造 / 缺失"三档 — 输出一张 1 页路线图。
- **延伸**: 同日 PH 第 16 [getviktor.com](https://getviktor.com/) (▲128) "AI coworker that actually does the work" — 也是 agent 当雇员/采购方叙事, 同主题独立信号。
